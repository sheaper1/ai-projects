# REVIEW.md — чек-лист ревью блока/секции (для критика и человека)

> Загружается во время ревью. Детерминированное уже проверено гейтом
> (`npm run check`: запретные файлы, `php -l`, `tokens:scan`, `lint:blocks`).
> Здесь — **судейские** правила, которые требуют сверки с Figma и глазами.
> Флагуй ТОЛЬКО то, что влияет на корректность/совпадение с макетом. Не
> придумывай улучшения «по вкусу» (анти-оверинжиниринг).

## Всегда проверять (vs Figma-нода секции)

1. **Значения из Figma, не на глаз.** Цвета/размеры/отступы совпадают с
   `get_design_context` / `get_variable_defs`. Сырьё → токен (`npm run tokens`).
2. **Переносы строк `<br>`** в заголовках/подписях стоят РОВНО как в макете
   (правило §16). Не автоперенос.
3. **Ширины.** Контент в центрированном контейнере 1344px (`__inner`), секция не
   растянута на весь экран. `vw`/`vh` — только в Hero.
4. **Адаптив по мобильному макету** (375px, поля 16px), а не выдуманные брейкпоинты.
   Нет системного UI, кривых высот, разъехавшегося ритма карточек.
5. **Картинки/иконки — из Figma.** Фото — `.webp`; иконки — SVG через медиатеку
   (media ID), не inline-`<svg>`, не PNG. Боксы не пустые.
6. **Hover на каждой кнопке** (правило памяти button-hover-rule).
7. **Репитер** для повторяющегося контента (add/remove/↑↓), а не захардкоженные
   N карточек.
8. **Карусели** — только `RbCarousel` (drag + бесконечный цикл), не самописные.
9. **Живой контент** не ломает адаптив: высоты от контента/сетки, не жёсткие px.

## Стиль (мягкие замечания)

- Дубли CSS, которые место в глобальных стилях темы на токенах.
- Reuse: подходил ли существующий блок вместо нового (`npm run blocks`).

## Пропускать (НЕ флагать)

- То, что уже ловит детерминированный гейт (хардкод-цвета, save, scss-import).
- Косметику без влияния на совпадение с макетом.
- Фоновый шум визуального диффа (рендер шрифтов, живой контент длиннее макета).
