Ты собираешь страницу **$ARGUMENTS** проекта Rosenberger из Figma. Источник дизайна
и правила — в репозитории. Иди СТРОГО по этим шагам, не пропускай стоп-гейты.
Цель — попасть в макет один-в-один, как на immobilie-verkaufen / immobilie-vermieten.

## 0. Контекст (прочитать ДО кода)
- Прочитай `AGENTS.md` — особенно **§6a (Playbook качества)**, §4 (золотые правила),
  §5 (токены), §10 (карта страниц + Figma fileKey), §11 (грабли).
- Прочитай хвост `CHANGELOG.md` и индекс памяти `MEMORY.md`. Сделай `git pull`.
- Figma-файл и node-id страницы (desktop + mobile) возьми из таблицы §10 по
  названию «$ARGUMENTS». Если страницы нет в таблице — СТОП, спроси пользователя.

## 1. Карта секций (reuse-first)
- `get_metadata(fileKey, nodeId=<desktop>)` → список секций сверху вниз
  (огромный ответ сохрани в файл, парси python/jq, НЕ тяни в контекст).
- `npm run blocks` → сопоставь КАЖДУЮ секцию с существующим блоком по атрибутам.
- Составь таблицу «секция → блок (reuse / новый / форма)». Покажи её пользователю.
- **СТОП-ГЕЙТ:** если секция требует НОВОГО блока, формы (WPForms/калькулятор) или
  нестандартной логики — назови это явно и спроси, прежде чем городить. Не выдумывай
  сложные блоки молча.

## 2. Значения и ассеты (программно, НЕ на глаз)
Для каждой секции:
- `get_design_context(node)` — точные тексты, размеры, ссылки на ассеты.
- Курсив/переносы заголовков уточняй точечным `get_screenshot(textNodeId)` — не угадывай.
- Сырые `#hex`/`px` → `npm run tokens -- <значения>`.
- Фото → WebP (`sharp`, ~1920, q80) в `projects/rosenberger/media/`.
- Иконки → `download_assets(format=svg)` → **`npm run figma:icon -- <файл.svg>`**
  (чистит фон-холста и геометрию) → потом в медиатеку. Иконки НЕ PNG.
- Контент, которого нет в Figma (свёрнутые ответы FAQ и т.п.) — пиши черновик и
  ПОМЕЧАЙ как черновик для пользователя. Не выдавай за финал.

## 3. Сборка
- Переиспользуй блоки через атрибуты в новом `scripts/import-$ARGUMENTS.mjs`
  (образец — `scripts/import-immobilie-verkaufen.mjs`). Новый блок — только если §1
  подтвердил; собирай в `/library`, копируй в тему, синхронизируй обе копии.
- Бренд/сторонние цвета (золото звёзд, Google-G) — инлайном в `render.php`, не в SCSS.
- Анимации — CSS (аккордеон = `grid-rows 0fr↔1fr`), не JS-высота.
- `npm run build` → `npm run check` (должно быть 0 нарушений токенов) →
  `node scripts/deploy-stack.mjs` → `node scripts/import-$ARGUMENTS.mjs`.

## 4. Визуальная сверка — ГЛАВНЫЙ СТОП-ГЕЙТ
Для КАЖДОЙ секции:
- `npm run shot -- <url> --sel ".wp-block-library-<slug>" --name <slug>`
- Открой PNG (Read), сравни с нодой Figma (`get_screenshot`).
- **Не переходи к следующей секции, пока эта не совпала** (отступы, шрифты, цвета,
  переносы, иконки). «Почти» — не считается, чини.
- Мобайл: `npm run shot -- <url> --mobile --full` — сверь раскладку по mobile-ноде §10.

## 5. Завершение
- Строка в `CHANGELOG.md` → маленький коммит `[Claude] …` → `git push`.
- В конце отдельным сообщением перечисли, что ОСТАВЛЕНО пользователю на ревью:
  формы, новые блоки, черновики текстов/FAQ, slug кнопок (`/kontakt/`?), любые места,
  где макет был неоднозначен.

Не выдавай задачу за готовую, если хоть одна секция не прошла сверку §4 или есть
нерешённый стоп-гейт. Лучше честный блокер, чем «почти похоже».
