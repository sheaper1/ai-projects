# AGENTS.md — единые правила для ИИ-агентов (Claude и Codex)

Над проектом по очереди работают два ИИ. Этот файл — **единый вход**. Полные
архитектурные правила — в [`CLAUDE.md`](CLAUDE.md); здесь не дублируем их, а задаём
рабочий протокол и приоритеты. **Конфликт правил → `CLAUDE.md` главнее.**

## 0. Перед любой работой

1. `git pull` (стянуть работу второго ИИ).
2. `git status` — если есть чужие незакоммиченные изменения, **сначала пойми,
   чьи они**, и не перетирай. Потерять работу второго ИИ — недопустимо.
3. Прочитай свежий хвост [`CHANGELOG.md`](CHANGELOG.md): что сделано в прошлый раз.

## 1. Что именно делать

- Конкретную задачу ставит пользователь. Нет задачи — спроси одним сообщением и стой.
- Сверься с `library/blocks-registry.md` перед созданием блока: похожий есть →
  копируй его, нет → создавай и **допиши строку в реестр**.
- Сомнения в макете/числах — бери из Figma программно, не на глаз
  (файл `3AzuInZ4YD95cLiQgiD24W`). Если ссылка ведёт на страницу, а не на фрейм,
  API вернёт «nothing selected» — попроси у пользователя ссылку именно на фрейм.

## 2. Жёсткие технические правила (кратко; подробно — CLAUDE.md)

- Только **динамические** блоки (`render.php`), без статического HTML в `save`.
- Стили — **только** через токены `theme.json` (`var(--wp--preset--…)`,
  `var(--wp--custom--…)`). Захардкоженные цвета/шрифты/размеры запрещены даже в
  редакторских контролах — используй существующие пресеты.
- Новый повторяемый параметр → сначала токен в `theme.json`, потом использование.
- Макет защищён `templateLock: "all"`; клиент правит только текст/картинки.
- Иконки — отдельные SVG в медиатеке (media ID/URL + `MediaUpload`); inline SVG и
  icon-font запрещены, кроме служебных ассетов темы.

## 3. Завершение задачи (обязательный цикл)

Для любой задачи, меняющей код/тему/плагин/контент:

1. Проверка: `npm run build` + `php -l` изменённых PHP + при правке вида —
   визуальная проверка страницы.
2. Деплой: `node scripts/deploy-stack.mjs`, убедись, что staging отвечает.
3. **Допиши строку в `CHANGELOG.md`** (формат — см. шапку того файла).
4. Коммит только связанных файлов: `[Claude] …` или `[Codex] …`, кратко по-русски.
   Не коммить `.env`, `build/`, `node_modules/`, `.claude/`, `.figma-tmp/`, чужое.
5. `git push` и проверь, что удалёнка приняла коммит.

Не прошло — не выдавай за готовое: чини или явно назови блокер.

## 4. Экономия токенов и меньше ошибок

Пользователь следит за расходом. Дёшево и точно:

- **Не перечитывай** файлы без нужды. Читай нужный фрагмент (offset/limit), не весь
  файл; для широкого поиска — Grep/Glob, а не чтение всего подряд.
- **Параллель**: независимые чтения/команды — одним батчем, не по одному.
- **Не пересказывай** уже известное и не описывай варианты, которые не делаешь.
  Решил — делай, не устраивай обзор.
- **Маленькие коммиты** по одной задаче — проще откатывать, меньше конфликтов.
- **Surgical changes**: трогай только нужное, чужой стиль не «улучшай».
- Перед большим изменением убедись, что рабочая версия закоммичена (точка отката).
